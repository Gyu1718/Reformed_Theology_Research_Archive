# UI Polish Notes

이 문서는 1차 UI 정리 패치의 기준과 구현 내용을 기록합니다.

## 1. 목적

자료 연결 기능이 늘어나면서 카드 안에 버튼과 보조 섹션이 많아졌습니다. 이 패치는 기존 데이터 구조와 `app.js`를 크게 바꾸지 않고, 보조 레이어에서 다음 문제를 완화합니다.

- 카드 과밀도
- 검색 후 복귀 어려움
- 버튼 스타일 파편화
- 본문 카드의 시각적 식별성 부족
- 특정 본문으로 바로 이동하는 링크 부족

## 2. 구현 파일

```txt
js/ui-polish.js
js/passage-detail-route.js
```

## 3. 주요 기능

### 검색 초기화 버튼

검색창 오른쪽에 `×` 버튼을 추가합니다. 검색어가 있을 때만 보이며, 클릭하면 검색어를 지우고 전체 목록을 다시 표시합니다.

### 필터 안내 표시

검색어가 있을 때 본문 영역 위에 현재 필터링 중임을 알려 주고, `전체 보기` 버튼을 제공합니다.

### 카드 보조 섹션 접힘 처리

카드 목록에서는 다음 보조 섹션을 접이식으로 정리합니다.

- 신학 연결
- 관련 책
- 관련 성경 본문
- 관련 역사

상세 페이지에서는 핵심 연결 섹션을 가능한 한 펼쳐 두고, 카드 목록에서만 세로 길이를 줄이는 방향을 우선합니다.

### 본문 직접 링크와 상세 페이지

다음 형식의 해시 링크를 지원합니다.

```txt
#passage=romans-9
#passage=john-1-14
#passage=2-timothy-3-16
```

`js/passage-detail-route.js`가 해당 해시를 감지하여 본문 상세 페이지를 렌더링합니다. 상세 페이지에는 본문 요약, 주제 태그, 신학 연결 메모, 관련 주제, 관련 역사, 관련 책, 핵심 질문, 전통별 활용, 설교 활용, 연구 활용이 표시됩니다.

### 성경 본문 카드 색상 보강

본문 카드는 개혁파/신정통주의 색상과 별도로 성경 본문 전용 색상 변수를 사용합니다.

```css
--scripture
--scripture-ink
--scripture-soft
```
